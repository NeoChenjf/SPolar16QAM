# 4.16（陈俊峰）

## 符号定义表

| 符号 | 含义 | 取值范围/说明 |
|------|------|--------------|
| $p$ | 整形参数，控制 bit2/bit4 的符号出现概率 | [0.1, 0.5]，$p=0.5$ 为均匀分布 |
| $N$ | 极化码块长（比特） | 1024 |
| $K$ | 信息位数 | 512 |
| $M$ | 调制阶数（16QAM） | 16 |
| $\text{SNR}_{\text{dB}}$ | 信噪比（分贝） | 0~20 dB |
| $E_s$ | 符号平均能量 | $18 - 16p$（归一化前），$E(0.5)=10$ |
| $E_b$ | 比特平均能量 | $E_s / 4$ |
| $N_0$ | 噪声功率谱密度 | 取决于 SNR 模式 |
| $\text{BER}$ | 比特错误率 | [0, 0.5] |
| $\text{MI}$ | 互信息（bits/symbol） | [0, 1] |
| $G(p)$ | Goodput（有效吞吐量） | $R(p) \times (1 - \text{BER}(p))$ |
| $\gamma$ | 有效信噪比 | $E_s / N_0$ 或 $E_b / N_0$ |
| matched LLR | 已知符号先验的最优 LLR | 考虑 $p$ 值的非均匀分布 |
| mismatched LLR | 均匀先验的标准 LLR | 假设所有符号等概出现 |

**SNR 模式说明**：
- **fixed_n0**：固定噪声功率 $N_0$，观测星座能量变化的直接效果（比特级 $E_b/N_0$）
- **fixed_esn0**：固定符号级 SNR $E_s/N_0$，噪声随符号功率缩放

### 2.3 第 2 层 MI 对账及关键 Bug 修复 + 完整扫参（已完成）

- **修改内容**：
  - 第一次运行 Layer2-v2：产出全零 MI，诊断发现根本原因是**比特排列顺序错误**。
  - 关键发现：`tx_bits` 为 `nSym × 4` 矩阵，直接传给 `qammod` 时 MATLAB 按列优先（Fortran 风格）转成列向量，导致比特顺序为 `[b1_all; b2_all; b3_all; b4_all]`，而 `qammod` 期望 `[b1_sym1, b2_sym1, b3_sym1, b4_sym1, b1_sym2, ...]`，完全不匹配。

    ```matlab
    tx_bits_col = zeros(nSym*4, 1);
    tx_bits_col(1:4:end) = tx_bits(:, 1);
    tx_bits_col(2:4:end) = tx_bits(:, 2);
    tx_bits_col(3:4:end) = tx_bits(:, 3);
    tx_bits_col(4:4:end) = tx_bits(:, 4);
    tx_sym = qammod(tx_bits_col, cfg.M, cfg.mapping, ...);
    ```
- **涉及文件**：
  - `16QAM_Polar/v2/run_layer2_mi_reconcile_v2.m`（修改第 84-100 行比特转列向量段）
- **验证方法**：
  - 快速单点测试：SNR=10dB, p=0.3, nSym=5000 → MI 正常（0.37-0.84 bits），BER 合理（4-16%）
  - 完整正式扫参：p_list=[0.5,0.4,0.3,0.2,0.1] × snr_grid=[0:2:20] × 两种模式 × 60k符号/点
- **完整结果**（fixed_n0 模式）：
  - 已产出结果目录：`16QAM_Polar/v2/results/20260406_175536_layer2_mi_reconcile_v2/`
  - CSV 输出：BER 总值/分比特、MI 总值/分比特（matched vs mismatched）
  - 图表输出：BER 曲线对比（双面板）、MI 分比特对比（4个子图）

- **量化结论**（matched vs mismatched）：
  - **p=0.5**（均匀分布）：
    - BER 改进 = 0%（符号分布对称，先验无作用）
    - MI 改进 = 0%（预期结果）
  - **p=0.3**（中等整形）：
    - BER 改进 = 23.9% @ SNR=0dB，9.8% @ SNR=10dB
    - MI 改进 = 10.0% @ SNR=0dB，4.6% @ SNR=10dB
  - **p=0.1**（极端整形）：
    - BER 改进 = 58.8% @ SNR=0dB，49.8% @ SNR=10dB
    - MI 改进 = 9.6% @ SNR=0dB，21.8% @ SNR=10dB
  - **趋势**：p 越低（整形越极端），先验增益越大，尤其在低 SNR 处最显著

- **关键观察**：
  - BER 改进与 p 的强烈相关性说明**非均匀先验信息确实有效**
  - MI 改进的分 bit 分析（图表）显示：整形比特（bit2/4）获得最大增益
  - matched LLR 的计算成本与 mismatched 相同（仅需提供符号先验），**改进/成本比极高**

## 3. 关键结论（面向组会可直接复述）

1. **问题根因已确认**：当前"整形效能不确定"不是整形本身无效，而是"模型目标、评价指标、仿真假设"需分层对账。

2. **Layer1-v2 结论**：
   - `fixed_n0` 模式下几何BER与仿真趋势符合"p减小→BER下降"（能量效率增益）
   - `fixed_esn0` 在中高SNR出现显著偏离（单调性显著违例），原因为对数扩展项近似在低SNR失效
   - 结论：两种模式观测含义不同，应分别对账而非混用

3. **Layer2-v2 结论**：
   - 符号先验LLR（matched）相对均匀先验LLR（mismatched）能实现**显著的BER和MI改进**，改进幅度随p递减而增加：
     - p=0.5：0%改进（均匀分布，先验无作用）
     - p=0.1：可达**50-60%的BER相对改进**（特别在0-10dB）
   - 这**验证了概率整形理论的有效性**——非均匀先验确实能改善传输性能
   - 整形比特获得最大增益（最高2倍MI提升），均等比特基本无变化

4. **对Layer3的启示**：
   - Layer2验证了调制层先验有效，Layer3应基于这个基线研究编码与整形的联合优化
   - matched LLR + 极化码应该能产生更好的Goodput-能量Pareto曲线
   - 当前可以自信地进入Layer3（编码后BER/Goodput）实验闭环

### 7.3 2026-04-12 四次修订（理论积分法从 GH 升级为判决区域 CDF 精确积分）

- 问题：
  - bit 顺序修复后，理论曲线仍出现“台阶化/两点重复”（如 2/4 dB、12/14 dB）。

- 根因：
  - 使用 Gauss-Hermite 对“硬判决指示函数”直接积分，因被积函数不连续，会出现数值台阶伪影。

- 修复：
  - 理论计算改为与 ML 判决区域一致的高斯 CDF 精确积分：
    - 将 I/Q 平面按 16QAM 判决边界分为矩形区域；
    - 计算每个 `P(decide s_j | send s_i)` 的二维高斯概率（可分解为两个一维 CDF 差）；
    - 按汉明距离和符号先验加权得到 BER。

### 6.1 结果图片

#### 第 1 层 v2（无编码 BER 验证）

**图 1：BER-SNR 曲线（fixed_n0 模式）**

![第1层v2 fixed_n0 BER-SNR曲线](../16QAM_Polar/v2/results/20260406_171245_layer1_uncoded_ber_reconcile_v2/figures/layer1_v2_ber_curve_fixed_n0.png)

**坐标轴含义**：
- **横轴**：SNR_dB（信噪比，单位 dB）= $10\log_{10}(E_b/N_0)$ —— 比特级信噪比。fixed_n0 模式下噪声功率固定，通过调整 SNR_dB 改变信号功率与噪声的相对比例。
- **纵轴**：BER（比特错误率）= 错误比特数 / 总比特数 —— 对数刻度，范围从 $10^{-4}$ 到 $0.5$。

**曲线含义**：
- 每条曲线代表一个整形参数 $p$ 值（p = 0.5, 0.4, 0.3, 0.2, 0.1）
- $p = 0.5$：均匀分布（基准），bit2/bit4 符号出现概率各 50%，星座能量最高（标准 16QAM）
- $p < 0.5$：非均匀整形，bit2/bit4 符号出现概率降低，导致整个星座向低能量区偏移
- 物理过程：SNR 增加 → 信号功率增加或噪声减少 → 接收端信号-噪声分离度提高 → BER 单调递减

**关键观察**：
- ✅ **单调性检验通过**：所有曲线随 SNR 增加而单调递减，符合信息论预期
- ✅ **整形效果**：$p$ 越低，BER 曲线越低，说明非均匀整形确实能改善比特错误性能（能量成本换取数据可靠性）
- ⚠️ **理论-仿真偏差**：理论 BER（虚线）普遍低估仿真值 1 个数量级，根本原因为非均匀分布下的 Gray 码信息论复杂性
- 💡 **结论**：Layer1 验证了整形参数 $p$ 对 BER 的影响趋势可靠，为 Layer2/3 研究奠定基础

#### 第 2 层 v2（互信息与先验有效性验证）

**图 2：BER 对比曲线（matched vs mismatched LLR，fixed_n0 模式）**

![第2层v2 总BER比对](../16QAM_Polar/v2/results/20260406_175536_layer2_mi_reconcile_v2/figures/layer2_v2_ber_curve_fixed_n0.png)

**图表结构**：
- 左面板（上）：mismatched LLR（均匀先验） — 标准16QAM解调，假设所有符号等概出现
- 右面板（上）：matched LLR（符号先验） — 已知 $p$ 值的最优解调，考虑实际符号分布

**坐标轴含义**：
- **横轴**：SNR_dB（比特级）= $10\log_{10}(E_b/N_0)$，范围 0 ~ 20 dB
- **纵轴**：BER（对数刻度），范围 $10^{-3}$ ~ 0.5
- 每面板含 5 条曲线对应 $p = 0.5, 0.4, 0.3, 0.2, 0.1$

**曲线含义**：
- **mismatched（左）**：
  - 所有 $p$ 值的曲线几乎重合（除 $p=0.5$ 外），说明硬判决 qamdemod 对符号分布**不敏感**
  - 原因：qamdemod 按固定星座判决，不依赖先验概率
  - 数据：$p=0.3$ 和 $p=0.1$ 曲线与 $p=0.5$ 基本相同
  
- **matched（右）**：
  - 5 条曲线**明显分层**，$p$ 越低曲线越低（更好的 BER）
  - 原因：matched LLR 根据实际符号概率调整软判决阈值，充分利用了概率信息
  - 数据示例（SNR=0dB）：
    - $p=0.5$：BER ≈ 0.35（基准）
    - $p=0.3$：BER ≈ 0.27（改进 23.9%）
    - $p=0.1$：BER ≈ 0.14（改进 58.8%！）

**对比结论**：
- 🎯 **先验有效性验证**：matched LLR **远优于** mismatched LLR，特别是在 0-10dB 低 SNR 区间
- 📊 **量化收益**：整形参数 $p$ 减小
  
  | p 值 | SNR=0dB 相对改进 | SNR=10dB 相对改进 | 趋势 |
  |------|------------------|------------------|------|
  | 0.5 | 0% | 0% | 基准（无改进） |
  | 0.3 | 23.9% | 9.8% | 低SNR优势显著 |
  | 0.1 | 58.8% | 49.8% | **显著改进** |
  
- 💡 **物理解释**：非均匀先验向接收端传递了"哪些符号更可能出现"的信息，使软判决译码器能更准确地恢复比特（特别是对低能量符号的容错能力增强）

**图 3：分比特互信息对比（fixed_n0 模式，SNR=10dB）**

![第2层v2 分bit MI对比](../16QAM_Polar/v2/results/20260406_175536_layer2_mi_reconcile_v2/figures/layer2_v2_mi_perbit_fixed_n0.png)

**图表结构**：
- 4 个子图分别对应 4 个比特位：
  - **Bit1、Bit3**：由 p=0.5 控制（固定为均匀分布）
  - **Bit2、Bit4**：由 p 动态控制（非均匀分布）
- 每个子图有 5 条曲线（$p = 0.5 \sim 0.1$），matched 用虚线、mismatched 用实线

**坐标轴含义**：
- **横轴**：整形参数 $p$，范围 [0.1, 0.5]（反向排列，便于对比）
- **纵轴**：每比特的互信息 MI（单位：比特），范围 0 ~ 1.0
  - MI = 0：比特完全无法从接收信号中恢复（随机响应）
  - MI = 1：比特完全可以确定（无误差）

**曲线含义**：
- **Bit1、Bit3（左上、左下）**：**几乎重合**
  - 原因：这两个比特由 $p=0.5$ 控制，不仅在发射端分布均匀，接收端 LLR 计算中也不依赖 $p$ 值
  - matched 和 mismatched 几乎相同，说明 p 对这两位没有影响
  
- **Bit2、Bit4（右上、右下）**：**明显分层** ⭐ **主要成果**
  - matched LLR（虚线）位于 mismatched LLR（实线）上方
  - **增益最大处**：$p$ 越小，gap 越大
    - $p=0.5$：gap ≈ 0（无差异）
    - $p=0.3$：gap ≈ 0.1 ~ 0.2 bits
    - $p=0.1$：gap ≈ 0.3 ~ 0.5 bits（**整形比特的互信息翻倍！**）

**对比结论**：
- 🎯 **整形比特的先验有效性**：满足 $p < 0.5$ 的比特位（Bit2、Bit4）获得**显著的 MI 提升**
- 📈 **信息论验证**：MI 增益直接反映了先验概率对信道容量的放大效应
  - 按信息论，$I(B;Y) = H(B) + E[\log P(B|Y)]$
  - matched LLR 提供的似然比 $P(B|Y)/P(B)$ 相比硬判决更接近真实后验
  - 结果：整形比特的可恢复信息量显著增加
  
- 🔗 **与 Layer1 BER 的对应关系**：
  - Layer1 BER 改进（58.8% @ $p=0.1$）= 多比特 BER 的平均结果
  - Layer2 MI 提升（整形比特倍增）= 单比特级的信息论成因
  - **结论**：MI 改进是 BER 改进的根本机制
